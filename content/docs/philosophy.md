# Philosophy

GoStarter is opinionated about architecture, not restrictive about product code.

## Explicit over magic

You should be able to trace a request from route to service to query without hidden frameworks.

## Server-first UI

Templ + Datastar are used to keep interactive flows simple while avoiding unnecessary SPA complexity.

## Production defaults

Security headers, role-aware routing, billing sync, and queue separation are treated as baseline concerns.

## Keep what works

The starter is designed to evolve into your product, not be replaced after a prototype phase.
